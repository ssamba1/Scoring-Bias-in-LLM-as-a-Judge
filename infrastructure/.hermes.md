## Project Identity

This is an LLM-as-a-Judge scoring bias research project investigating how **format bias** (sensitivity to presentation structure) and **content bias** (sensitivity to answer quality) behave across model scales and training paradigms (base vs instruct, SFT vs RLHF).

## Key Findings (must be correct in all outputs)

- **Format bias ↓ after instruction tuning** — ROBUST across ALL 8 model families, ALL sizes (0.5B–7B).
- **Content bias ↑ only in 3B+ RLHF models** — scale-dependent differential effect. ≤1.5B SFT models show uniform bias reduction.
- **IIAR rejected** at 0.5B and 3B — replaced by Format Efficiency Hypothesis.
- 8 base-instruct families × 22 instruct models tested.

## Workflow Rules

1. **Always run tests before declaring done** — `cd research-draft && python -m pytest tests/ -x -q` or `python tests/run_all.py`
2. **No `print()` in production code** — use logger
3. **Prefer `pathlib.Path`** over `os.path`
4. **Checkpoint early, checkpoint often** — platform path differences WILL bite if you don't
5. **Cost gates** — never spin up a Colab run without checking if a Kaggle T4 run suffices
6. **Verify model IDs and auth** before giving instructions — check HF_TOKEN, provider caps, model availability
7. **Never fabricate results** — if a run failed or didn't produce output, report the failure
8. **Run `make lint` before committing** — catches formatting and style issues
9. **Run `make test` to verify nothing is broken** after any code change
