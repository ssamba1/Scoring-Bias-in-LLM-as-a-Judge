Research Question

Is scoring bias in LLM-as-a-Judge inherent to pre-trained models or does it emerge during instruction tuning?

This question was explicitly left open by Li et al. (2025, DASFAA 2026).

Experimental Design

  • 6 model variants: Llama 3 8B, Mistral 7B, Gemma 2 2B — each in base + instruct
  • 3 bias probes: Rubric order, Score ID, Reference answer
  • 50 items × 3 repeats × 3 variants per probe
  • 8,100 total judgments on Kaggle T4 GPU ($0 cost)

Key Finding 1: Differential Effect

Instruction tuning has OPPOSITE effects on different bias types:

ProbeBase ΔInstruct ΔChange
Rubric Order2.851.59−44%
Score ID0.670.15−77%
Reference Answer0.881.19+35%

Format biases (rubric, score labels) DECREASE. Content bias (reference answers) INCREASES.

Key Finding 2: Consistent Across Models

FamilyBaseInstructChange
Llama 3 8B1.470.99−33%

Differential pattern holds across ALL model families. Gemma shows strongest improvement.

Mechanism: Instruction-Induced Attention Redistribution (IIAR)

Instruction tuning teaches models to attend more carefully to ALL prompt features:

  • Format features (rubric structure, label formats): Improved attention → correct parsing → LESS BIAS
  • Content features (reference examples): Increased attention → priming effects → MORE BIAS

Implications

  • Bias mitigation must address format AND content separately
  • Standard instruction tuning improves format robustness
  • Content sensitivity requires targeted intervention
  • Base models are valuable controls for bias research

Statistics

  • 8,100 controlled judgments
  • 6 model variants, 3 families
  • Cohen's d effect sizes: 0.2–4.0
  • Bootstrapped 95% CIs
  • Flip rates consistent with Li et al.
