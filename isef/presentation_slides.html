The Problem

LLMs are used as automated judges to evaluate other AI systems. But they're biased.

  • 35+ bias types documented
  • LLM-as-a-Judge used in Chatbot Arena, MT-Bench, AlpacaEval
  • If judges are biased, model rankings are unreliable
Slide 2 / 12

The Gap

Li et al. (2025, DASFAA 2026) documented scoring bias across 5 models, but explicitly stated:

"The underlying causes of these scoring biases remain to be validated."

Nobody has investigated whether scoring bias comes from pre-training or instruction tuning.

Why This Happens: IIAR Hypothesis

Instruction-Induced Attention Redistribution:

  • Instruction tuning → models attend more to ALL prompt features
  • Format features (rubric, labels) → correctly parsed → LESS BIAS
  • Content features (examples) → priming effects → MORE BIAS

Elegantly explains the differential effect.

Slide 8 / 12

Implications

  • For mitigation: Must target format AND content separately
  • For practice: Report bias profiles alongside scores
  • For research: Include base model baselines
  • Key insight: Blanket "debiasing" will backfire — improving format robustness doesn't fix content sensitivity
Slide 9 / 12

Limitations

  • 50 items — sufficient for significance, fewer than benchmarks
  • 2B–8B models only — larger models may differ
  • Can't distinguish SFT vs RLHF
  • English-only results
  • Single seed (42), single prompt template
  • No human baseline yet
