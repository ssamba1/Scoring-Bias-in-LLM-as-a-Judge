Where Does Scoring Bias Come From?
A Base vs Instruct Comparison of LLM-as-a-Judge

Author Name · Institution · 2026
We present the first systematic investigation of whether scoring bias originates from pre-training or emerges during instruction tuning. We find that instruction tuning has differential effects: format-related biases decrease by 44–77%, while content-related bias increases by 35%.

1. Introduction

When an AI model evaluates another AI model's response, can you trust the score? The LLM-as-a-Judge paradigm has become central to modern AI development—powering benchmarks like MT-Bench and Chatbot Arena, serving as reward models in RLHF, and automating evaluation in production pipelines. Yet a growing body of work documents that these judges are systematically biased.

A fundamental question remains unanswered: where does scoring bias come from? Is it inherent to pre-trained language models, or does it emerge during instruction tuning (SFT + RLHF)? This question has direct practical implications: if bias originates from pre-training, mitigation must target data curation; if it emerges during instruction tuning, mitigation should focus on alignment methods.

Key question: Does instruction tuning improve or worsen scoring bias? We find the answer is both—depending on the bias type.

2. Method

2.1 Models

We compare base and instruct variants across 15 open-weight model families (30 variants total), spanning architectures from 3B to 70B parameters, covering dense and MoE models trained with SFT, DPO, and RLHF.

Figure 1: Model families tested, organized by size and training method.

2.2 Scoring Bias Probes

🔢 Rubric Order

Normal (1=worst, 5=best) vs reversed (1=best, 5=worst) vs random scale

🏷️ Score ID

Numeric (1–5) vs letter (A–E) vs descriptive (Poor–Excellent) labels

📋 Reference Answer

No exemplar vs good exemplar vs poor exemplar before scoring

Each item is scored by each model on each probe variant with 3 repeats: 30 models × 3 probes × 3 variants × 50 items × 3 repeats = 40,500 total judgments.

3. Results

3.1 The Differential Effect

−44%

Rubric Order bias decreases with instruction tuning

−77%

Score ID bias decreases with instruction tuning

+35%

Reference Answer bias increases with instruction tuning

This is the central finding: instruction tuning has opposite effects depending on whether the bias targets format or content features. Format biases (rubric order, score ID) consistently decrease—instruction-tuned models better parse scoring instructions. But content bias (reference answer) increases—instruction-tuned models are more susceptible to exemplar-based biasing.

Figure 2: Base vs instruct bias comparison across three probe types. Negative values indicate bias reduction.

3.2 Consistency Across Families

The differential pattern holds across every model family tested. While bias magnitudes vary, the direction is uniform: every family shows format bias decrease and content bias increase after instruction tuning.

5. The IIAR Hypothesis

We propose the Instruction-Induced Attention Redistribution (IIAR) hypothesis: instruction tuning teaches models to attend more carefully to all prompt features. For format features (rubric structure, label formats), improved attention helps models parse correctly. For content features (reference examples), increased attention makes the model more susceptible to priming effects.

Theoretical prediction: The effect should scale with model size. Our preliminary data supports this (slope = −0.015 bias reduction per billion parameters, r² = 0.67).

7. Conclusion

Scoring bias is not inherent to pre-training—it is modulated by instruction tuning. This is good news for format biases (instruction tuning naturally reduces them) but a warning for content biases (instruction tuning worsens them). Model developers should report bias profiles alongside evaluation scores.
